# Support VelantisWind

> Open-source wind energy workflows, directly inside QGIS.

VelantisWind is a free and open-source QGIS plugin designed to support early-stage wind farm analysis, validation workflows and geospatial pre-assessment tasks.

The project brings together wind energy calculations and GIS-based workflows in a single QGIS environment, helping users work with layouts, wind resource data, wake effects, noise assessment and shadow flicker analysis.

VelantisWind is designed to be transparent, adaptable and useful as a complement to existing technical workflows.

---

## Why support VelantisWind?

VelantisWind is built around a simple idea:

> Wind energy tools should be more transparent, accessible and adaptable to real project workflows.

The project aims to provide an open-source foundation that can be tested, improved and validated with the support of users, consultants, developers, researchers, universities and technical partners.

Your support helps transform VelantisWind from an open-source plugin into a more robust, documented and validated tool for practical wind energy workflows.

## Adaptation to client workflows

VelantisWind can also be adapted to specific technical needs.

Every company, consultant or project team works slightly differently. Some workflows require specific input files, custom reports, internal validation procedures, particular GIS layers, additional constraints or different output formats.

Because VelantisWind is open source and built inside QGIS, it can be extended and adapted to fit real project workflows.

Possible adaptations include:

| Need                      | Example                                                             |
| ------------------------- | ------------------------------------------------------------------- |
| Custom reports            | Company-specific PDF, CSV or Excel outputs                          |
| Input formats             | Adaptation to internal GIS, turbine or wind resource formats        |
| Validation workflows      | Comparison with internal calculations or reference project data     |
| GIS integration           | Custom QGIS layers, styling, templates and project structures       |
| Environmental constraints | Additional exclusion areas, receptors, buffers or regulatory layers |
| Technical modules         | New calculations or improvements requested by users                 |
| Pilot projects            | Testing VelantisWind on real layouts and real wind resource data    |

Workflow adaptations, technical collaborations and project-specific developments can be discussed separately from general sponsorship.
