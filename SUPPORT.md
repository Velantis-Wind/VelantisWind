# Support VelantisWind

> Open-source wind energy tools, directly inside QGIS.

VelantisWind is a free and open-source QGIS plugin designed to support practical wind farm pre-assessment workflows.

It brings together geospatial analysis and wind energy calculations in a single QGIS-based environment, helping users evaluate layouts, energy production, wake effects, noise and shadow flicker at early project stages.

---

## What VelantisWind offers

| Area                  | Purpose                                                                       |
| --------------------- | ----------------------------------------------------------------------------- |
| Energy assessment     | Estimate Annual Energy Production from wind resource data and turbine layouts |
| Wake-aware evaluation | Analyse layout performance considering wake effects                           |
| Noise assessment      | Support early-stage acoustic impact analysis                                  |
| Shadow flicker        | Estimate potential shadow flicker impact around wind turbines                 |
| GIS workflow          | Work directly inside QGIS with layers, rasters, layouts and project data      |
| Custom adaptation     | Adapt inputs, outputs and reports to real technical workflows                 |

---

## Why support the project?

VelantisWind is built around a simple idea:

> Wind farm pre-assessment tools should be more accessible, transparent and adaptable.

Many wind energy workflows rely on expensive, closed or highly specialized software. VelantisWind follows a different approach: an open-source foundation that can be tested, improved, validated and adapted with the support of users, consultants, companies, universities and researchers.

Your support helps transform VelantisWind from a promising open-source tool into a robust technical platform for real wind energy workflows.

## Adaptation to client workflows

VelantisWind can also be adapted to specific technical needs.

Every company, consultant or project team works slightly differently. Some workflows require specific input files, custom reports, internal validation procedures, particular GIS layers, additional constraints or different output formats.

Because VelantisWind is open source and built inside QGIS, it can be extended and adapted to fit real project workflows.

Possible adaptations include:

| Need                      | Example                                                             |
| ------------------------- | ------------------------------------------------------------------- |
| Custom reports            | Company-specific PDF, CSV or Excel outputs                          |
| Input formats             | Adaptation to internal GIS, turbine or wind resource formats        |
| Validation workflows      | Comparison with WAsP, WindPRO, OpenWind or internal tools           |
| GIS integration           | Custom QGIS layers, styling, templates and project structures       |
| Environmental constraints | Additional exclusion areas, receptors, buffers or regulatory layers |
| Technical modules         | New calculations or improvements requested by users                 |
| Pilot projects            | Testing VelantisWind on real layouts and real wind resource data    |
